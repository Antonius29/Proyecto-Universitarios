# Sistema de Gestión de Clientes (SGC) - Descripción del Proyecto

## Enfoque del Proyecto

El Sistema de Gestión de Clientes (SGC) es una solución integral diseñada para que empresas de desarrollo de software puedan centralizar y administrar de manera eficiente todos los aspectos de sus relaciones comerciales y proyectos.

## Fin Principal

El objetivo principal de este proyecto es proporcionar una plataforma única y centralizada donde una empresa de software pueda:

1. Gestionar todos sus clientes en un solo lugar
2. Mantener registro de contactos clave en cada empresa cliente
3. Administrar múltiples proyectos simultáneamente
4. Asignar y dar seguimiento a tareas de desarrollo
5. Catalogar sus productos y servicios
6. Almacenar y organizar toda la documentación relacionada

## Problema que Resuelve

Antes de este sistema, las empresas de software enfrentaban:

- Información de clientes dispersa en múltiples lugares
- Pérdida de documentos importantes
- Dificultad para dar seguimiento a múltiples proyectos
- Falta de visibilidad sobre el estado de las tareas
- Comunicación desorganizada con clientes
- Duplicación de esfuerzos

## Solución Proporcionada

El SGC soluciona estos problemas ofreciendo:

- Base de datos centralizada con toda la información de clientes
- Organización clara de proyectos por cliente
- Descomposición de proyectos en tareas asignables
- Almacenamiento seguro de documentos
- Visualización rápida del estado de todo
- Acceso controlado mediante autenticación

## Beneficios Clave

PARA LA EMPRESA:
- Mayor control sobre sus proyectos
- Mejor organización de recursos
- Reducción de tiempo en búsqueda de información
- Mejora en la entrega de proyectos
- Mayor profesionalismo ante clientes

PARA LOS EQUIPOS DE DESARROLLO:
- Claridad sobre qué tareas asignadas tienen
- Acceso a documentación del proyecto
- Comunicación centralizada

PARA LOS CLIENTES:
- Mayor transparencia en el estado de sus proyectos
- Acceso a documentación y especificaciones
- Mejor servicio y comunicación

## Uso Típico del Sistema

1. Un cliente nuevo contacta a la empresa
2. Se registra el cliente en el sistema
3. Se registran los contactos clave del cliente
4. Se crea un proyecto vinculado al cliente
5. El proyecto se desglosa en tareas específicas
6. Se asignan tareas a miembros del equipo
7. Se sube toda la documentación necesaria
8. El equipo actualiza el estado de las tareas
9. Se genera seguimiento sobre el progreso
10. Se marca el proyecto como completado

## Módulos del Sistema

El sistema está compuesto por 6 módulos principales:

CLIENTES: Registro y administración de empresas que contratan servicios

EQUIPO: Gestión de contactos y responsables dentro de cada empresa cliente

PROYECTOS: Creación y seguimiento de proyectos de desarrollo

TAREAS: Desglose de proyectos en tareas específicas y asignación de equipo

PRODUCTOS: Catálogo de servicios y soluciones que ofrece la empresa

DOCUMENTOS: Almacenamiento centralizado de especificaciones, contratos, diseños, etc.

## Tecnología Utilizada

Backend: PHP con arquitectura MVC
Base de Datos: MySQL con procedimientos almacenados
Frontend: HTML5, CSS3, JavaScript vanilla
Patrón: Tres capas (Presentación, Negocio, Datos)
